Code Smell — Lazy Class (Classe Preguiçosa)

Equipe: Ana Júlia Santinoni, Janaina Carlos

Definição (resumo)

Lazy Class (Classe Preguiçosa) é uma classe que possui pouca ou nenhuma responsabilidade útil, normalmente armazenando apenas dados sem comportamento. Ela não justifica sua existência e costuma ser melhor mesclada com outra classe.

Consequências

  • Fragmentação do design: muitas classes pequenas e inúteis.
  • Aumento da complexidade sem benefício funcional.
  • Dificuldade de manutenção e navegação no código.
  • Risco de duplicação de lógica entre classes relacionadas.

Exemplos (Código com smell e refatoração)

JavaScript — Com Lazy Class


// Classe preguiçosa: apenas guarda dados, sem comportamento
class ClienteDados {
  constructor(nome, email) {
    this.nome = nome;
    this.email = email;
  }
}

class ClienteService {
  constructor(dados) {
    this.dados = dados;
  }

  enviarEmailBoasVindas() {
    console.log("Bem-vindo(a), " + this.dados.nome);
  }
}

JavaScript — Refatorado


class Cliente {
  constructor(nome, email) {
    this.nome = nome;
    this.email = email;
  }

  enviarEmailBoasVindas() {
    console.log("Bem-vindo(a), " + this.nome);
  }
}

Java — Com Lazy Class


// Endereco é apenas um container de dados (lazy)
public class Endereco {
  private String rua;
  private String cidade;

  public Endereco(String rua, String cidade) {
    this.rua = rua;
    this.cidade = cidade;
  }

  public String getRua() { return rua; }
  public String getCidade() { return cidade; }
}

Java — Refatorado


public class Cliente {
  private String nome;
  private String rua;
  private String cidade;

  public Cliente(String nome, String rua, String cidade) {
    this.nome = nome;
    this.rua = rua;
    this.cidade = cidade;
  }

  public void mostrarEndereco() {
    System.out.println(nome + " mora em " + rua + ", " + cidade);
  }
}

Python — Com Lazy Class


class PedidoInfo:
    def __init__(self, numero, valor):
        self.numero = numero
        self.valor = valor

class PedidoService:
    def __init__(self, pedido_info):
        self.pedido_info = pedido_info

    def imprimir(self):
        print(f"Pedido {self.pedido_info.numero}, valor {self.pedido_info.valor}")

Python — Refatorado


class Pedido:
    def __init__(self, numero, valor):
        self.numero = numero
        self.valor = valor

    def imprimir(self):
        print(f"Pedido {self.numero}, valor {self.valor}")

Como resolver / refatorar

  1. Identificar classes com responsabilidades mínimas (poucos atributos, sem métodos).
  2. Avaliar se a classe pode ser inline (mesclada) em outra que tenha comportamento.
  3. Se a classe for parte de uma hierarquia com subclasses preguiçosas, considerar collapse hierarchy.
  4. Remover classes obsoletas ou transformar em estruturas de dados simples quando for adequado.
